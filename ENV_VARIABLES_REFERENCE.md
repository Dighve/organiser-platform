# Environment Variables - Quick Reference

## Render Dashboard - Manual Setup Required

After deploying with Blueprint, add these environment variables in the Render dashboard:

### üîê Cloudinary (REQUIRED for image uploads)

```bash
CLOUDINARY_CLOUD_NAME=xyz
CLOUDINARY_API_KEY=678..
CLOUDINARY_API_SECRET=w3456..
```

**Where to add:**xs
1. Render Dashboard ‚Üí hikehub-backend service
2. Environment tab ‚Üí Add Environment Variable
3. Add each variable above
4. Click "Save Changes"

---

### üìß Email (OPTIONAL - for password reset, notifications)

```bash
RESEND_API_KEY=<your-resend-api-key>
EMAIL_FROM=onboarding@resend.dev
```

**Note:** App works without email. Skip if you don't have Resend account.

---

### üåê Frontend URL (Pre-configured, update if needed)

```bash
FRONTEND_URL=https://hikehub-poc.netlify.app
```

**Update if:** You deploy frontend to a different Netlify URL

---

## Auto-Configured Variables (DO NOT MODIFY)

These are set automatically by Render:

```bash
DATABASE_URL=<auto-populated-from-database>
JWT_SECRET=<auto-generated-random-string>
SPRING_PROFILES_ACTIVE=prod
JAVA_TOOL_OPTIONS=-Xmx512m
PORT=<auto-set-by-render>
```

---

## Netlify Dashboard - Frontend Setup

Add this environment variable in Netlify:

```bash
VITE_API_URL=https://hikehub-backend.onrender.com
```

**Where to add:**
1. Netlify Dashboard ‚Üí Your site
2. Site configuration ‚Üí Environment variables
3. Add `VITE_API_URL` with your Render backend URL
4. Save and trigger redeploy

**Important:** Replace `hikehub-backend.onrender.com` with your actual Render service URL

---

## Common Issues

### ‚ùå Images not uploading
**Check:** Cloudinary variables are set correctly
**Fix:** Verify credentials, redeploy backend

### ‚ùå CORS errors
**Check:** FRONTEND_URL matches your Netlify URL
**Fix:** Update FRONTEND_URL, redeploy backend

### ‚ùå 401 Unauthorized errors
**Check:** JWT_SECRET is set
**Fix:** Should be auto-generated, check Render dashboard

### ‚ùå Database connection failed
**Check:** DATABASE_URL is populated
**Fix:** Verify database service is "Available"
